## Challenges

- **Story 4.4's HIGH finding was a spec oversight.** AC5 required "no session saved *or restored*" — `session::save()` was correctly gated, but `session::load()` was not. The story spec explicitly said "No changes to `session.rs`", so the developer faithfully followed the spec and the load-side gap wasn't caught until code review. Same root cause as the Epic 2 CPU loop: a wrong spec produces a correct implementation of wrong behaviour.

- **Test-purity issue from 4.3 was not fixed in 4.4.** `test_reset_session_clears_state` and `test_reset_session_is_undoable` call through `App::dispatch` → `session::save()` → file write to `~/.rpncalc/session.json` during `cargo test`. Story 4.3 flagged this as a future fix for 4.4. It was not addressed.

- **Integration test placeholders: fourth deferral.** On the action list since Epic 1's retro. The project closed with `session_roundtrip::session_integration_placeholder` and `keybinding_sequences::placeholder` still as stubs. The project has 325 unit tests and zero real integration tests.

---

## Key Lessons Learned

1. **The spec-error pattern is systemic.** Epic 2: wrong event loop pattern in Dev Notes. Epic 4: incomplete AC missing the load side of a save/load pair. Both followed the same path — spec error → faithful wrong implementation → review catch → fix cycle. Mitigation: add a SM pre-spec checklist specifically for symmetric API pairs (save/load, read/write, encode/decode) to verify both sides are covered in ACs before the story is written.

2. **Test-purity debt compounds.** Tests that write to the user's home directory are fragile in CI, surprising in clean environments, and hard to reason about. Any test calling through to I/O should use path injection or a test-only config. The fix was known in 4.3, deferred to 4.4, not done.

3. **Integration test placeholder debt ran the entire project.** Four epics, zero real integration tests. 325 unit tests provide strong coverage at the function level but no seam-level verification. This was the single item carried unresolved on every retro's action list.

4. **When the process works, it really shows.** Story 4.2's clean review, story 4.3's APPROVED with only an INFO, zero debug log entries across the epic — these reflect four epics of accumulated discipline. The system works. The remaining gaps are known and named.
