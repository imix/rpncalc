## Successes

- **Architecture held.** Typed dispatch (`Op` enum), atomic operations (stack unchanged on error), `CalcState` as sole mutable state, no raw string errors anywhere in the final codebase.
- **Test discipline was excellent.** 131 tests at end of a pure-engine epic. Every error path covered. Atomicity rule (stack unchanged on `Err`) tested religiously.
- **Scope control was strong.** Each story deferred what didn't need to land (e.g. HexStyle display → Story 3.3, TUI wiring → Epic 2). No scope creep.
- **Story 1.3 was a model execution.** Zero debug issues — the Dev Notes were complete enough that implementation matched spec exactly.
- **The `FBig` footgun was caught by tests both times it appeared.** The safety net worked.

---

## Challenges

- **No working software delivered in Epic 1.** `main.rs` still prints "Hello, world!". The FRs are covered at the engine level but zero users can do anything with this binary today. This was a deliberate trade-off — not a failure — but a trade-off worth naming.
- **`FBig::to_string()` binary string bug appeared twice** (Stories 1.2 and 1.4) despite being documented after the first occurrence. Mitigation: tests caught it both times; the display path now has explicit tests.
- **Story 1.5 scope was underestimated in Dev Notes.** "Exactly two files" was actually three (`error.rs` needed `PartialEq`). The story File List correctly captured reality, but the Dev Notes note was stale.

---

## Key Lessons Learned

1. **Always have something working.** Even a minimal CLI REPL in `main.rs` (read line → parse → compute → print) would have proven the engine end-to-end from Story 1.2 onward. Working software is the north star at every step, including engine stories.

2. **Tests are the safety net for known gotchas.** The `FBig` binary string issue is a real footgun, but as long as display paths have tests, it gets caught. Document the gotcha, test the output, move on.

3. **Atomicity discipline pays off.** The peek-compute-then-mutate pattern in `binary_op` / `unary_op` was extra effort in Story 1.4 but it means Epic 2 can trust the engine completely. Never partially mutate state.
