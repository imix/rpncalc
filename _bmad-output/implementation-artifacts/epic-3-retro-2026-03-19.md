## Challenges

- **Alpha mode hints were underdelivered.** The 3-line static list (`Enter push`, `Esc cancel`, `Bksp delete`) is a placeholder. The full `AlphaSubmitThen` bindings (`+`, `-`, `*`, `/`, `^`, `%`, `!`, `n`, `s`, `d`, `p`, `r`) that push-and-execute in one keystroke are completely undiscoverable.
- **STORE discoverability was forgotten.** Story 3.5 surfaces existing register RCL commands, but there is no hint anywhere for how to *create* a register. The STORE syntax (`myvar STORE`) is invisible to a new user.
- **Current store UX is awkward.** Workflow is `3 ENTER i myvar ENTER` — no dedicated store key, no hints guiding the user to it.
- **Integration test stubs remain placeholders.** Deferred from Epic 1 retro, deferred again. Now a third-time item.

---

## Key Lessons Learned

1. **Review all three hints pane states during story creation.** UX-DR4 defines three states (Normal, Alpha, Chord). Alpha state was scoped as 3 lines in Story 3.1 and never revisited. Future stories touching hints pane must explicitly validate all three states.

2. **Discoverability stories must verify the capability is actually discoverable.** Story 3.5 added register recall to the hints pane but never addressed how registers get created in the first place. A discoverability AC should cover the full user journey, not just the feature being implemented.

3. **Pre-solved Dev Notes are worth the investment.** Every story in this epic executed cleanly. Zero debug log entries across 6 stories is directly attributable to having the implementation pre-worked in Dev Notes before development started.
